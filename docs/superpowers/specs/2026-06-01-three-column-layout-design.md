# Спецификация дизайна: Трехколоночный макет и Глобальные цели синхронизации

Этот документ описывает переработку веб-интерфейса AgentSync Web GUI: перевод вкладки Agent Manager на трехколоночный макет и внедрение централизованного управления целями синхронизации (активными агентами) в шапке приложения.

## 1. Функциональные требования

1. **Глобальные цели синхронизации**:
   * Перенос выбора целевых AI-агентов для синхронизации из панели деталей отдельных компонентов в шапку приложения.
   * Добавление выпадающего списка (Dropdown с чекбоксами) рядом с кнопкой **Smart Merge Deploy**.
   * Поддержка выбора трех агентов:
     * Claude Code (CLI) (`claude-code`)
     * Claude Desktop (`claude-desktop`)
     * Antigravity IDE & CLI (`antigravity`)
   * Персистентное сохранение выбранных целей в файле `manifest.yaml` на сервере.

2. **Трехколоночный макет во вкладке Agent Manager**:
   * Левая панель: Выбор категории компонентов (MCP Серверы, Правила, Навыки, Рабочие процессы, Хуки).
   * Центральная панель: Список всех доступных компонентов выбранной категории с отображением названия и краткого описания.
   * Правая панель: Детали выбранного компонента. Кнопки синхронизации файлов источника (`Записать локально`, `В глобальный каталог`).

## 2. Архитектура и изменения

### Backend (Go)

1. **Модель данных и хранилище (`store.go`)**:
   * Добавить метод `SaveManifest(manifest Manifest) error` для перезаписи файла `manifest.yaml` в корне репозитория с сохранением структуры YAML.
   ```go
   func (s *Store) SaveManifest(manifest Manifest) error {
       manifestPath := filepath.Join(s.RepoPath, "manifest.yaml")
       data, err := yaml.Marshal(&manifest)
       if err != nil {
           return err
       }
       return os.WriteFile(manifestPath, data, 0644)
   }
   ```

2. **HTTP API Маршруты (`web.go`)**:
   * Добавить GET-обработчик `/api/manifest` для чтения текущего манифеста и возврата списка `active_agents` в формате JSON.
   * Добавить POST-обработчик `/api/manifest` для обновления списка `active_agents` в манифесте.
   * Зарегистрировать новые эндпоинты в функции `StartWebServer`:
     ```go
     http.Handle("/api/manifest", corsMiddleware(http.HandlerFunc(apiManifestHandler(repoPath))))
     ```

### Frontend (SolidJS & CSS)

1. **Управление состоянием (`App.jsx`)**:
   * Добавить сигналы `activeAgents` (массив выбранных ID агентов) и `showDropdown` (булево состояние открытия меню).
   * При монтировании компонента вызывать функцию `fetchManifest` для загрузки активных агентов с сервера.
   * При изменении чекбокса отправлять POST-запрос на `/api/manifest` с обновленным списком агентов и логировать операцию.

2. **Компонент шапки (`App.jsx`)**:

## 3. План верификации

### Ручное тестирование
1. Запустить веб-интерфейс, проверить вкладку **Agent Manager**:
   * Левая панель должна переключать категории.
   * Центральная панель должна корректно выводить список компонентов.
   * Выбор компонента должен открывать его детали в правой панели.
   * Наличие кнопок синхронизации источников в деталях.
2. Проверить выбор целей синхронизации в шапке:
   * Открыть выпадающий список.
   * Переключить чекбокс (например, отключить `Antigravity IDE & CLI`).
   * Проверить в терминале лог об обновлении манифеста и убедиться, что файл `manifest.yaml` переписался без ошибок.
   * Нажать **Smart Merge Deploy** и убедиться по логам консоли, что деплой был выполнен только для выбранных (активных) агентов.
